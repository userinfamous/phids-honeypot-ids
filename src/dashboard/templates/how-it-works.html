How PHIDS Works

Understanding the PHIDS honeypot system: architecture, attack simulation, and data collection

System Overview

What is PHIDS?

PHIDS (Python Honeypot Intrusion Detection System) is an educational honeypot system that simulates vulnerable services to observe and analyze attack patterns. It's designed for learning, testing, and understanding cybersecurity threats in a controlled environment.

Attack Simulation

Important: Simulated Attacks

PHIDS generates simulated attack traffic for educational purposes. These are NOT real attacks from actual threat actors. The system is designed to demonstrate how honeypots work and how attacks are detected and logged.

HTTP Honeypot

Simulates a vulnerable web server on port 8081. Accepts weak credentials and logs all authentication attempts.

SSH Honeypot

Simulates an SSH server on port 2222. Accepts valid credentials and provides a fake shell environment.

Data Collection

All connection attempts, authentication events, and commands are logged to a SQLite database for analysis.

IP Address & Geolocation

How IP Addresses Work

When you test the honeypot, the system logs your actual IP address. For simulated attacks, the system can generate IP addresses from different ranges. Geolocation data is simulated based on IP ranges for educational purposes:

  • 1-50: Simulated as United States
  • 51-100: Simulated as Europe
  • 101-150: Simulated as Asia
  • 151-200: Simulated as Various
  • 201-255: Simulated as Global

Attack Types Simulated

HTTP Attacks

Weak Credentials: admin:admin, admin:password, root:root, test:test, and more
Result: Successful login redirects to dashboard success page

SSH Attacks

Valid Credentials: root:password, admin:admin, user:user, test:test, ubuntu:ubuntu
Result: Successful login provides fake shell access with command logging

Success vs. Failed Attacks

Successful Attack

  • Uses credentials from weak list
  • Status: SUCCESS
  • Session ID generated
  • Commands logged (SSH)
  • Full forensic data available

Failed Attack

  • Uses credentials NOT in weak list
  • Status: FAILED
  • No session ID
  • No commands logged
  • Minimal forensic data

Data Flow

Attack Attempt ↓ Honeypot Service (HTTP/SSH) ↓ Authentication Check ↓ Log to Database ↓ Dashboard Real-time Update ↓ Analyst Review & Analysis

Testing the System

Transparency & Educational Purpose

Educational System

PHIDS is designed for educational purposes. All data is simulated or locally generated. The system helps you understand:

  • How honeypots detect and log attacks
  • How authentication attempts are tracked
  • How geolocation analysis works
  • How to analyze security events
  • Best practices for threat intelligence
