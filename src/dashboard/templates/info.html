What is PHIDS?

PHIDS (Python Honeypot IDS) is a comprehensive cybersecurity education and research platform that simulates vulnerable services to attract and analyze cyber attacks in real-time.

Purpose & Mission

  • Cybersecurity Education: Hands-on learning for students and professionals
  • Threat Research: Real-world attack pattern analysis and documentation
  • Security Awareness: Demonstrate attack techniques and defense strategies
  • Incident Response Training: Practice detecting and responding to threats

Security Assessment & Recommendations

Important Security Notice

PHIDS is designed for educational and research purposes. This system intentionally exposes vulnerable services and should only be deployed in controlled, isolated environments.

Network Isolation Status

Current Protections
  • Dashboard bound to localhost (127.0.0.1:5001)
  • Container-based isolation
  • Non-root user execution
  • Resource limits configured
Security Considerations
  • Honeypots exposed on 0.0.0.0 (all interfaces)
  • No authentication on dashboard
  • Unencrypted data storage
  • No rate limiting implemented

Security Hardening Recommendations

Immediate Actions (Critical)
  1. Network Isolation: Deploy in isolated VLAN or DMZ
  2. Dashboard Authentication: Implement user authentication
  3. HTTPS: Enable TLS encryption for web interface
  4. Firewall Rules: Restrict access to management interfaces
Enhanced Security (Recommended)
  1. Database Encryption: Encrypt SQLite database at rest
  2. Rate Limiting: Implement connection rate limits
  3. IP Filtering: Geographic and network-based restrictions
  4. Monitoring: Add security monitoring and alerting

System Architecture & Data Flow

Data Flow Process

1. Attack Initiation

External attackers target the honeypot services (SSH on port 2222, HTTP on port 8081) believing they are legitimate vulnerable systems.

2. Honeypot Interaction

Honeypots capture all interaction data including commands, payloads, authentication attempts, and connection metadata with precise timestamps.

3. Real-time Analysis

The IDS Engine analyzes captured data using signature-based detection to identify attack patterns and generate security alerts.

4. Data Storage

All events are stored in the SQLite database with comprehensive logging including attack success analysis and geolocation data.

5. SOC Monitoring

The web dashboard provides real-time monitoring with WebSocket updates, allowing security analysts to observe attacks as they happen.

6. Export & Integration

Data can be exported in CSV format for integration with SIEM systems and further analysis.

Deployment Guide

Docker Deployment (Recommended)

# Quick Start with Docker Compose
git clone https://github.com/userinfamous/phids-honeypot-ids.git
cd phids-honeypot-ids
docker-compose up -d

# Access Dashboard
http://localhost:5001

Manual Installation

# Install Dependencies
pip install -r requirements.txt

# Run PHIDS
python main.py

# Run Dashboard Only
python start_dashboard.py

Configuration

Key configuration options in config.py:

# Honeypot Services
HONEYPOT_CONFIG = {
    "ssh": {"port": 2222, "enabled": True},
    "http": {"port": 8081, "enabled": True},
    "ftp": {"port": 2121, "enabled": False}
}

# Dashboard Settings
DASHBOARD_CONFIG = {
    "host": "127.0.0.1",  # Change for external access
    "port": 5001,
    "enabled": True
}
Deployment Warnings
  • Educational Use Only: Not suitable for production without security hardening
  • Network Isolation: Deploy in isolated network segments
  • Legal Compliance: Ensure compliance with local laws and regulations
  • Monitoring Required: Implement proper monitoring and incident response
