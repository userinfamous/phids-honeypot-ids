Manual Testing Guide

Learn how to manually trigger attacks against the honeypot. These examples show you exactly how the system works.

HTTP Successful Login

Test successful authentication with weak credentials

curl -X POST http://127.0.0.1:8081/admin \ -H "Content-Type: application/x-www-form-urlencoded" \ -d "username=admin&password=admin" \ -L

Expected: 302 redirect to success page with session ID

import requests response = requests.post( 'http://127.0.0.1:8081/admin', data={'username': 'admin', 'password': 'admin'}, allow_redirects=False ) print(f"Status: {response.status_code}") print(f"Redirect: {response.headers.get('Location')}")

Expected: Status 302 with Location header

  1. Open http://127.0.0.1:8081/admin in your browser
  2. Enter username: admin
  3. Enter password: admin
  4. Click Login
  5. Expected: Redirect to success page with session information
HTTP Failed Login

Test failed authentication with strong credentials

curl -X POST http://127.0.0.1:8081/admin \ -d "username=admin&password=SuperSecurePassword123!" \ -L

Expected: 200 OK with login form (no redirect)

SSH Successful Login

Test successful SSH authentication

ssh root@127.0.0.1 -p 2222 # When prompted, enter password: password whoami pwd exit

Expected: Successful authentication with shell access

SSH Failed Login

Test failed SSH authentication

ssh root@127.0.0.1 -p 2222 # When prompted, enter password: WrongPassword123

Expected: Authentication failed message

How to Verify Results
  • Dashboard: Check the main dashboard for new connection events
  • Database: Query data/phids.db for connection records
  • Logs: Check log files for authentication events
  • Status: Look for SUCCESS (green) or FAILED (red) indicators
